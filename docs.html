Verify Webhook Signature

Use this recipe when you need to confirm that an incoming request was really sent by your documentation system or integration provider. The steps walk through parsing the signature, computing the expected hash, and sending the payload only after the check succeeds.

  1. Parse the signature header.
  2. Sign the request body with the shared secret.
  3. Compare the computed signature with the header.
  4. Continue only if the values match.
Open in New Tab
Node 
const expected = crypto
  .createHmac("sha256", process.env.WEBHOOK_SECRET)
  .update(`${timestamp}.${JSON.stringify(body)}`)
  .digest("hex");
